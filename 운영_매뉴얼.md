# 울산대학교 총무인사팀 챗봇 운영 매뉴얼

> 이 문서는 챗봇을 처음 인수받은 담당자를 위한 운영 가이드입니다.

---

## 1. 챗봇 개요

울산대학교 총무인사팀 업무 매뉴얼을 기반으로 직원들의 질문에 자동 답변하는 AI 챗봇입니다.

- **서비스 URL**: Streamlit Cloud에 배포 (담당자에게 URL 확인)
- **AI 모델**: Claude Haiku (Anthropic)
- **소스 코드**: https://github.com/dongldh/chatbot

---

## 2. 구조 이해

```
chatbot/
├── docs/          ← 원본 업무 매뉴얼 (MD 파일)
├── wiki/          ← AI가 정리한 위키 페이지 (챗봇이 실제로 읽는 파일)
├── streamlit_app.py   ← 챗봇 앱
├── wiki_builder.py    ← 위키 생성 로직
└── build_wiki.py      ← 위키 빌드 실행 스크립트
```

**핵심 개념:**
- `docs/` = 원본 문서 보관함 (직접 수정 가능)
- `wiki/` = AI가 docs를 읽고 정리한 결과물 (직접 수정 금지, 항상 빌드로 생성)
- 챗봇은 `wiki/`의 내용을 바탕으로 답변함

## 3. 필수 계정 및 키

| 항목 | 용도 | 위치 |
|------|------|------|
| Anthropic API Key | Claude AI 호출 | console.anthropic.com |
| Google 서비스 계정 | 질문 로그 기록 (Google Sheets) | Google Cloud Console |
| GitHub 계정 | 소스코드 관리 및 배포 | github.com/dongldh/chatbot |
| Streamlit Cloud | 웹 서비스 호스팅 | share.streamlit.io |

**Streamlit Cloud 시크릿 설정 위치:**
`Streamlit Cloud 대시보드 → 앱 선택 → Settings → Secrets`

```toml
ANTHROPIC_API_KEY = "sk-ant-..."
OPENAI_API_KEY = ""         # 현재 미사용
ADMIN_PASSWORD = "비밀번호"
SHEET_URL = "https://docs.google.com/spreadsheets/..."

[gcp_service_account]
# Google 서비스 계정 JSON 내용
```

## 4. 새 문서 추가하는 방법

새로운 업무 매뉴얼을 챗봇에 학습시키는 전체 과정입니다.

### Step 1. MD 파일 준비
- 업무 내용을 마크다운(.md) 형식으로 작성
- **주의**: 이미지(`![[image.png]]`) 는 챗봇이 읽을 수 없으므로 텍스트로 대체
- 파일 크기는 2MB 이하 권장

### Step 2. docs/ 폴더에 추가
로컬에서 직접 복사:
```bash
cp 새문서.md /chatbot/docs/
```
또는 GitHub에서 직접 업로드 (`docs/` 폴더 → Add file)

### Step 3. 위키 재구축
```bash
python3 build_wiki.py
```
- 소요 시간: 약 3~5분 (문서 수에 따라 다름)
- 비용: 약 $0.30 (문서 34개 기준)
- Claude Haiku가 전체 문서를 분석해 wiki/ 폴더를 새로 생성

### Step 4. GitHub에 Push
```bash
git add docs/ wiki/
git commit -m "새 문서 추가: 문서명"
git push origin main
```

### Step 5. 배포 확인
- GitHub push 후 Streamlit Cloud가 자동으로 재배포 (1~3분 소요)
- 서비스 URL에서 질문해서 반영 여부 확인

## 6. 비용 관리

**월 예상 비용 (현재 규모 기준):**

| 항목 | 비용 |
|------|------|
| 챗봇 답변 (Haiku) | 100회 질문 ≈ $0.10~0.15 |
| 위키 재구축 1회 (Haiku) | ≈ $0.30 |
| Streamlit Cloud 호스팅 | 무료 플랜 사용 중 |

**크레딧 충전:** console.anthropic.com → Plans & Billing → Add credits
- 최소 충전: $5
- 권장 충전: $10~20 (수개월치)

**크레딧 부족 시 증상:** 챗봇에 질문하면 에러 발생

## 7. 관리자 패널 사용법

챗봇 화면 왼쪽 사이드바에서 접근합니다.

1. **관리자 비밀번호 입력** (Streamlit Secrets의 `ADMIN_PASSWORD`)
2. 기능:
   - **MD 파일 업로드**: 새 문서를 직접 업로드
   - **위키 재구축**: 업로드 후 반드시 실행 (1~2분 소요)
   - **원본 문서 관리**: 파일 목록 확인 및 삭제
   - **질문 로그 보기**: Google Sheets에 기록된 질문/답변 확인

> ⚠️ 관리자 패널에서 위키를 재구축해도 GitHub에 자동 반영되지 않습니다.
> Streamlit Cloud 재시작 시 초기화될 수 있으므로 로컬에서 빌드 후 push하는 방법을 권장합니다.

## 8. 자주 발생하는 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 챗봇 답변 시 에러 | Anthropic 크레딧 부족 | console.anthropic.com에서 충전 |
| 새 문서 내용이 반영 안 됨 | 위키 재구축 미실행 | `python3 build_wiki.py` 실행 후 push |
| 앱이 열리지 않음 | Streamlit Cloud 슬립 모드 | 페이지 새로고침 (1~2분 대기) |
| 위키 빌드 중 오류 | API 크레딧 부족 또는 네트워크 오류 | 크레딧 확인 후 재실행 |

## 9. 로컬 개발 환경 설정

처음 개발 환경을 구성할 때 필요한 과정입니다.

```bash
# 1. 저장소 클론
git clone https://github.com/dongldh/chatbot.git
cd chatbot

# 2. 의존성 설치
pip3 install -r requirements.txt

# 3. 환경변수 설정 (.env 파일 생성)
ANTHROPIC_API_KEY=sk-ant-...

# 4. 앱 실행
python3 -m streamlit run streamlit_app.py
```

## 10. 문서 작성 팁

챗봇이 잘 답변하려면 MD 파일을 이렇게 작성하세요.

**좋은 예:**
```markdown
## 연차 신청 방법
1. UWIN 접속
2. 근태관리 → 휴가신청 메뉴 선택
3. 날짜 및 사유 입력 후 [신청] 버튼 클릭
4. 팀장 승인 후 확정
```

**나쁜 예:**
```markdown
아래 그림 참조
![[image123.png]]
```

- 절차는 번호 목록으로 작성
- 시스템 메뉴 경로를 구체적으로 명시
- 이미지 대신 텍스트로 설명
- 주의사항은 별도로 명시
